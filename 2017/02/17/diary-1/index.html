闲谈一个初级程序员在初次入职的感受

1 代码命名

1.1 数据库命名

1.1.1 统一大小写（我赞成全部大写） 多个单词用下划线分割。

如：销售人员表 
正确示例：SALES_PERSON/sales_person 
错误示例：salesPerson

1.1.2 命名清楚

  • 意思表达清晰：能表达出此 数据库/数据表/列 的含义。
  • 反对使用拼音命名。毕竟中华字典太过于博大精深，容易造成含义不清楚。若英文不好的同学，花两秒查查谷歌翻译也比用拼音强。

1.2 程序代码命名

  • 参数变量/类名/方法名等采用驼峰命名法。
  • 终态命名，即在 JAVA 中使用 final 修饰的采用：统一大写，多单词使用下划线分割。
  • 命名清晰：
    • 避免使用数字区分同一类型的变量。如person1/person2。
      意思表达清晰：能表达出此 参数/方法/类等 的含义，不要使用拼音命名。

2 代码格式

  • 避免将代码写到一整行。一行代码过长即不美观，也不方便阅读。

  • 适当使用空行。将不同逻辑的隔开，方便阅读。

  • 代码缩进。一般缩进四个字符。

  • 相关IDE格式快捷键：eclipse：CTRL+F IDEA:CTRL+ALT+L

3 代码内容

3.1 此处我想说的是：关于异常的那些事

        避免使用 try…catch… ，使用 throw… ，谁的错就还给谁。
然后在前台页面将错误信息展示给用户，告诉 TA 犯的错。

3.2 消除代码冗余

并不是代码写的越多就越好。
如下面程序片段：

01> if (state != null) {
02> 
03>     if (state == 1) {
04> 
05>         hql += " and wp.state = 1 ";
06> 
07>     }

08>     if (state == 0) {

09>     hql += " and wp.state = 0 ";

10>     }
11> }

第三行到第十行完全可以使用一行代替：

hql += " and wp.state =  " + state;

若有人认为 state 只有0/1取值，合并的一行就不能过滤0/1。我想此问题完全可以加入最外层判断语句中。

3.3 创建变量的位置

哈，看到此，有人就会问变量位置有啥好说的，创建到哪不都一样，能用就行？

先不多说，看如下程序片段：

01>  List<People> peoples = new ArrayList<>();
02>  
03>  for(int i = 0; i<10; i++){
04>      People people = new People();
05>      peoples.add(people);
06>  }

在第四行会造成频繁的创建变量。倘若100次，1000次，10000次呢？内存就会不停的创建，销毁。

若将此代码片段稍微改动一下：

01>  List<People> peoples = new ArrayList<>();
02>  People people = null;
03>  
04>  for(int i = 0; i<10; i++){
05>      people = new People();
06>      peoples.add(people);
07>  }

这样的话，就不会重复的开启内存空间喽。

3.4 代码的复用

        将自己的代码写的有价值。代码的复用同样是消除冗余代码的一部分。在此多多利用 JAVA 中三大特性之一的封装。

4 拥有自己的一套代码

        身为程序员，一直都无法拥有属于自己的代码，岂不是很失落？我们虽不能与大神相比，但是我们可以写上一套代码供自己重复使用。

        比如说：用户模块。基本上每个网站都有需要用户模块，因此，这部分，我们完全可以单独拿出来，并将此中的相关内容：权限验证，数据库表等完善。
这样就有了属于自己的一代程序代码。况且 IDE 都可以将代码程序打包成 JAR 文件，这样，以后就方便我们使用了。有空的话，还可以维护更新，这样
也能提高自己。
